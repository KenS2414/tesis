# Release v0.1.0-stabilization

Fecha: 2026-03-24

## Resumen
Este release establece una linea base estable del proyecto tras la migracion a arquitectura orientada a app factory y la alineacion de runtime y CI.

## Cambios principales
- Contrato de arranque centrado en `create_app()`.
- Eliminacion de compatibilidad legacy en imports del modulo principal.
- Runtime alineado entre Docker/Compose y systemd/nginx.
- Carril de integration CI endurecido con Postgres y MinIO reales.
- Clarificacion del naming de pruebas S3 (fake, funcional e integration real).
- Consolidacion documental operativa en `docs/runbook.md`.

## Validacion
- `pytest -q -m "not integration"`: 48 passed, 1 skipped, 2 deselected.
- `pytest -q -m integration`: 2 passed, 1 skipped, 48 deselected.

## Notas operativas
- En despliegues con systemd, tras cambios de unidad aplicar:
  - `systemctl daemon-reload`
  - `systemctl restart gunicorn`
